Problem 1 – Movie Rating

Description
Company X is building out a new tool that allows users to browse the movies that their company makes. As part of this tool we’d like to allow users to rate the movies so that we can begin to build a recommendation engine. Your job is to build a page that will display a list of movies to a user and allow them to rate the movies.

Requirements
The page should retrieve a list of movies from a URL. The data will be returned as JSON – please request the data asynchronously as in the future we might wish to add filters to the page to allow the user to filter the list in real time. Your page should display the list of movies with applicable metadata.

In addition to the list of available movies, please provide a way to rate each movie. You can choose how to format the message you send to the server with the rating (both URL structure & message structure). The server will provide one of two responses to your rating with a 33% failure rate:

{ status: “SUCCESS” }
{ status: “FAILURE”, reason: “sample reason” }

Make sure you let the user know whether their rating was successful or not!

API

Method Endpoint Description
GET /hiring/list.php Get a list of movies in the Company X system
? /hiring/rate.php Rate a movie in the Company X system
